Overview

DomainIntel API aggregates domain intelligence from multiple authoritative sources into a single, consistent JSON API:

  • RDAP — registration status, key dates, nameservers, registrar (via rdap.org)
  • DNS — A, AAAA, NS, CNAME, SOA, MX, TXT, CAA records (via DNS-over-HTTPS)
  • Certificates — certificate transparency logs (via crt.sh)
  • Bulk — parallel multi-domain, multi-type lookups in one request

All responses are cached (RDAP 24h, DNS 5m, certificates 1h) and rate limited per client. A successful response uses the envelope {"data": ..., "cached": false}.

Authentication

When deployed behind the RapidAPI gateway, include your proxy secret header:

X-RapidAPI-Proxy-Secret: YOUR_SECRET

For direct access (outside RapidAPI), use an API key instead:

X-API-Key: YOUR_API_KEY

If neither header is present, rate limiting falls back to client IP address.

Endpoints

Health Check

GET/v1/health

Returns service health. Not rate limited.

curl https://YOUR_API_HOST/v1/health \
  -H "X-RapidAPI-Proxy-Secret: YOUR_SECRET"

Response:

{ "status": "ok" }

RDAP Lookup

GET/v1/rdap/{domain}

Registration Data Access Protocol lookup. Returns domain status, key dates (registration, expiration, last changed), nameservers, and registrar.

curl https://YOUR_API_HOST/v1/rdap/example.com \
  -H "X-RapidAPI-Proxy-Secret: YOUR_SECRET"

DNS Lookup

GET/v1/dns/{domain}?type={type}

DNS record lookup over HTTPS. Supported types: A, AAAA, NS, CNAME, SOA, MX, TXT, CAA. Defaults to A.

curl "https://YOUR_API_HOST/v1/dns/example.com?type=MX" \
  -H "X-RapidAPI-Proxy-Secret: YOUR_SECRET"

Certificate Transparency

GET/v1/certificates/{domain}?limit={N}&match={mode}

Certificate transparency logs from crt.sh. Records are sorted by issuance date (newest first) and include associated domain names (name_value).

ParamDefaultDescription
limit50Max records to return (1–500).
match—Set to wildcard to include subdomain certificates.
curl "https://YOUR_API_HOST/v1/certificates/example.com?limit=10&match=wildcard" \
  -H "X-RapidAPI-Proxy-Secret: YOUR_SECRET"

Bulk Lookup

POST/v1/bulk

Query up to 20 domains in parallel across one or more lookup types. Each sub-query reuses the cache. Individual failures are reported per-domain without blocking the rest.

curl -X POST https://YOUR_API_HOST/v1/bulk \
  -H "X-RapidAPI-Proxy-Secret: YOUR_SECRET" \
  -H "Content-Type: application/json" \
  -d '{
    "domains": ["example.com", "google.com"],
    "types": ["rdap", "dns"],
    "dns_type": "A"
  }'

Response (abbreviated):

{
  "data": [
    {
      "domain": "example.com",
      "rdap": { "domain": "example.com", "registrar": "..." },
      "dns": { "domain": "example.com", "type": "A", "records": [...] }
    },
    {
      "domain": "google.com",
      "rdap": { ... },
      "dns": { ... }
    }
  ],
  "cached": false
}

Error Responses

All errors use a consistent envelope:

{
  "error": {
    "code": 404,
    "message": "domain not found",
    "type": "not_found"
  }
}
HTTP StatusTypeMeaning
400bad_requestMalformed request or invalid parameter.
404not_foundDomain not found in upstream registry.
429rate_limitedRate limit exceeded (includes Retry-After).
502upstream_errorUpstream provider (RDAP/DNS/crt.sh) error.
500internal_errorUnexpected internal error.
